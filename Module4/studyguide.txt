1. The Big Three
	1.1. WHat are the Big Three?
		- Copy Constructor: Creates a new object as a copy of an existing one
			- Syntax: ClassName(const ClassName& other)
		- Copy Assignment Operator: Copies one existing object into another existing object
			- Syntax: ClassName& operator=(const ClasssName& other)
		- Destructor: Cleans up when an Object dies
			- Syntax: ~ClassName()
	1.2. What is the "rule of three"?
		- The rule of the big three is that if you use dynamically allocated memory you most likely need
		  the implement the big three. Also if you have implemented one of the big three you most likely
		  need to implement the other two. 
	1.3. When are the members of the Big Three called or used?
		- Copy Constructor: When a brand new object is created from an existing object
			- Remember this: Brand New Objecy + Another Object as Source = Copy Constructor
				- Example: ClassName obj1; ClassName obj2 = obj1; // Copy Constructor is called
		- Copy Assignment Operator: when an existing object is overwritten by another existing object
			- Remember This: Existing Object + Another Object as Source = Copy Assignment Operator
				- Example: ClassName obj1; ClassName obj2; obj2 = obj1; // Copy Assignment Operator is called
		- Destructor: When an object goes out of scope or is explicitly deleted
	1.4. Why must you write the members of the Big Three?
		- Copy Constructor: 
			- If you dont assign a copy constructor, the compiler will generate a shallow copy, a double free, and a dangling pointer. 
				- Shallow copy: same memory address, no new allocation
				- Double Free: When two objects point to the same memory, and both are destroyed, leading to a double free and the second "free" will crash because the memory was already free.
				- Dangling Pointer: When you free the memory but a pointer still exist pointing to where the memory use to be.
		- Copy Assignment Operator:
			- If you dont assign a copy assignment operator, the compiler will generate a shallow copy, a double free, and a dangling pointer. 
				- Shallow copy: same memory address, no new allocation
				- Double Free: When two objects point to the same memory, and both are destroyed, leading to a double free and the second "free" will crash because the memory was already free
				- Dangling Pointer: When you free the memory but a pointer still exist pointing to where the memory use to be.
		- Destructor:
			- If you dont assign a destructor, the compiler will generate a default destructor that does not free up any dynamically allocated memory, leading to memory leaks.
				- Memory Leak: When you lose the reference to dynamically allocated memory without freeing it, leading to wasted memory and potential performance issues.
	1.5: What are the similarities between a copy constructor and a copy assignment operator?
		- Both take "const ClassName& other" as a parameter
		- & to avoid infinite recursion
		- const to promise not to modify the original
		- To do a deep copy
		- New memory allocation
		- And copying the data from the source to the new memory

2. Dynamic Memory in Classes
	2.1. What is a Shallow Copy?
		- When a dupliate pointer points to the same memory address as the original pointer.
	2.2. What is a Deep Copy?
		- When new allocated memory is created for a new pointer to point to and the data from the original memory address is copied to the new memory address.
	2.3 What is a dangling pointer?
		- a dangling pointer is when the memory address is freed and the pointer still points to the freed memory.
	2.4. In the absence of any other information, what should you initalize pointers to?
		- Nullptr: because a dangling or uninitalized pointer points to some random garbage memory address.
		- If you accidently dereference it you get undefined behavior, crashes, and currupted data. 

3. The const Keyword
	3.1 What does the const keyword stand for?
		- Const stands for constant, it means whenever you slap const before a variable, you cannot modify it after its been set
	3.2. What does it mean to label variable as const, and why would you want to do that?
		- Labeling a variable as const:
			- Safety: Prevents accidental modification
			- Intent: Tells other programmers that this value should never change.
			- Compiler catches mistakes: instead of silent bugs at  runtime
		- Labeling a classname with const:
			- Protects original data from accidental changes.
	3.3. What does it mean to return a const type from a function, and why would you want to do that?
		- It promises that the caller cannot modify the returned value, and that outside methods cannot modify data and change it. 
	3.4. What does it mean to create a const class member function, and why would you want to do that?
		- protects the data insde the class
		- The function itself promises not to modify any member variables
