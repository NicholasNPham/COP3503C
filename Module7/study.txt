FILES I/O IN GENERAL

What does a full path of a file include?
- The full filepath requires 3 things
	- Drive/root (like C:\ or /)
	- Directory path (the folder leading to the files)
	- Filename (including the extensions)
- So something like C:\Users\Nick\Documents\data.csv 
	- Drive, folder, filename all together thats the full path

What is a delimiter, and what purpose does it serve in a file?
- A delimiter is just a character used to separate pieces of data
- In files you'll see commas, spaces, tabs, pipes
	- Whatever the format uses.

What is a CSV File
- CSV: Comma Separated Values 
	- It is a plain text file that separates values with commas.
	- One record per line typically
	- Excel just happens to be a common whay to view them

True or false: All formats are the same
- FALSE
- Different formats have different structures, rules and purposes.
- A .csv uses comma-delimited text, a .json uses key-value pairs, a .png is binary image data, and a .docx is a zipped collection of XML files
- You cannot treat them interchangably

What's the difference between a forward slash and a backslash when it comes to file paths?
- /: Forward Slash
- \: Back Slash
- They are both used for file paths and the backslash specifically (\) for escape characters
	- /: The forward slash is universal, works on Windows, Mac, and Linux
	- \: the backslash is Windows-specific for file paths
	- \: also doubles as the escap character in C++, which is what \n, \t, \\ etc.
	
FILE I/O IN PRACTICE

What classes can we use to open files for reading and writing?
- first think of the prefixes
	- i = input (reading)
	- o = output (writing)
	- f = file

- The class we use to open file for reading is
	- i = input (reading)
	- f = file
	- ifstream = input (reading) file stream

- the class we use to open file for writing is
	- o = output (writing)
	- f = file
	- ofstream = output (writing) file stream

What operators can we use to read and write data to and from a file?
- ofstream uses << to write (just like cout)
- ifstream uses >> to read tokens (just like cin)
	- >> reads one token at a time
		- It stops at whitespaces (spaces, newlines, tabs)
		- So if you file has "stringname intnumber"
			- doing file >> name >> age grabs them separately

How can you read an entire line from a file?
- getline(fileStream, stringVariable) reads the whole line including spaces, stops at the newline.
- getline has two forms
	- getline(stream, variables);
	- getline(stream, variables, delimiters);
- stream: the thing you're reading from
	- could be an ifstream, or even a cin, or an istringstream
- variable, a string where the results gets stored
- delimiter (optional): the character to stop at
	- defaults to \n (newline) if you leave it out
	- you pass in ',' to stop at commas instead

What is a data-driven program?
- A program that loads its data from an external source (like a file or csv) rather than hardcoding values directly into the source code
- The advantage is that when data changes, you update the file not the codebase.
- The program just reads whatever is there.

What are the differences between absolute paths and relative paths?
- Absolute Path: 
	- the full path from the root/drive all the way to thefile
	- Like C:\users\Nick\Documents\data.csv
	- It works from anywhre but it brittle
	- If you move the file or run it on a different machine it breaks
- Relative path
	- A path relative to wherever the program is currently running from.
	- Like just data.csv or data/data.csv.
	- No drive, no full directory chain

Where does a program look for files by default?
- by default a program will look for files in its working directory
	- Wherever it is currently being executed from
- the program looks in where the exe lives the Debug/Release folder

Where does an IDE look for files by default?
- by default the IDE will look for files in the project directory
- VS or the IDE will set the working directory to that project folder when you run it from the IDE
- it looks in the .vcxproj file

What is an escape sequence?
- An escape sequence is a \ (backslash) followed by a character that represents something you cant other type directly in a string literal.
- They are needed because certain characters (like newlines, tabs, or backslashes themselves) have no direct keyboard representation inside the string
- Common ones:
	- \n : newline
	- \t : tab
	- \\ : literal backslash
	- \" : literal quote inside a string

WHAT THE QUIZ WANTS

Create a a data-driven application.
Read data from a CSV file.

- EXAMPLE: STUDENT DATA

- THIS IS THE CSV
	
	Name,Grade,Age
	Nick,95,20
	John,88,21
	Sara,72,19

1. Create a struct for the csv/data.
	
	struct Student
	{
		string _name;
		int _grade;
		int _age;
	};

2. Create a loadFile function 

	void loadFile(vector<Student>& studentList, string filename)
	{
	}

3. The body of the function

- note: 
	- top of the function needs the follow in order to create this function
		- #include <fstream>
			- ifstream/ofstream
		- #include <sstream>
			- istringstream
		- #include <vector>
		- #include <string>
		

	void loadFile(vector<Student>& studentList, string filename)
	{
		
		ifstream csv(filename);
		
		if (!csv.is_open())
		{
			return;
		}

		string headerLine;
		getline(csv, headerLine);

		string lineFromDataFile;
		getline(csv, lineFromDataFile);

		while (!csv.fail())
		{

			istringstream ss(lineFromDataFile);

			string name;
			getline(ss, name, ',');

			string gradeString;
			getline(ss, gradeString, ',');
			int grade = stoi(gradeString);

			string ageString;
			getline(ss, ageString, ',');
			int age = stoi(ageString);

			Student studentObject;

			studentObject._name = name;
			studentObject._grade = grade;
			studentObject._age = age;

			studentList.push_back(studentObject);

			getline(csv, lineFromDataFile);
		}
		csv.close();
	}

Explain the purpose of escape sequences.
- The purpose of escape sequences is that the characters like ( \n \t \\ \" ) cannot be represented directly inside a string literal
- For example
	- You cant hit enter in the middle of the string, and have it mean new line.
		- so \n is the way to represent it in code

Explain the purpose of delimiters in files.
- The purpose of delimiters is to use a character to sparate values or data.
- In a CSV specifically that delimiter is a comma
	- WHich is how your program knows where one field ends and the next begins.
