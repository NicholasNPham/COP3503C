Lecture 1: Operator Overloading

- What are operators?
	- Things you've already used before: + / * += < !=
	- What are they, really?
	- Functions with operator in the name.
	- These functions are used to costomize the way you work with classes

- Overloading Functions
	- Function overloading is what allows you to create multiple versions of a function, wach with different signautres (different parameter list)

- Operators are shorthand for function
- What operators can you overload?
	- Arithmetic Operators, Relational Operators, and assignment and compound operators
- Overloaded operators should Match Expectations
	- Example Code:

		// Assignment
		int a, b = 2, c = 5;
		a = b + c; // combine b and c into a new integer

		// You should be able to catch the results
		result = thingA + thingB;

		// Examples: the + operator should not change things
		int A = b + c;  // don't change b or c
		int x = 5 + 12; // 5 stays 5, 12 stays 12
		b + x;			// result; Sums of b and x is discarded

	- This code, applied to primative types shoud have no effect ( and ytour comipler may even ignore it in the end)
	- For a class, the same behavior should be followed.
