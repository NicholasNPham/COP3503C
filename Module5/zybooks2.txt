10.8 Additional material: A first linked list

Overview
	- A common use of pointers is to create a list of items such that an item can be efficently inserted somewhere in the middle of the list, whithout the shifting of later itms as required for a vector.
	- The following program illustrates how such a list can be created.
	- A class is defined to represent each list item, kown as a list node.
	- A node is comprised of the data to be stored in each list item, in this case just one int,
	- And the pointer to the next node in the list.
	- A special node named head is created to represent the pront of the list, after which regular items can be inserted. 
	- An item can be inserted somehwere in the middle of the list without having to shift all subsequent items. 
	- The purpose of a list's head node is to provide a pointer to the first items node in the list if such an item exists

	The Steps in order for linkedList is:
		- Save the old next pointer
		- Point this to the new node
		- Point the new node to what used to be next 
	- If you do step 2 before 1m yoiu lose the chain.

	   this->babiesVal = babiesInit;

	   tmpNext = this->nextNodePtr;

	   return this->nextNodePtr;

12.8 Overloading stream operators
	- Overloading the << operator
		- The c++ << operator is kown as the insertion operator. 
		- A C++ class can overload the insertion operator by creating a member function named operator<<.
		- Answer Notes:
			- The << operator's implemetation uses case-sensitive string comparisons, so that all-lowercase strings is not recognized
			- If the operator << function return type were to changed to void the compiler will still compile and produce the same results when ran.
	- Overloading the >> operator
		- The C++ >> operator is known as the extraction operator.
		- A C++ class can overload the extraction operator by creating a member function named operator>>
		- The extraction operator does not add to the line but rather removes a string from the front of the line
		- No check for empty list exist, the extraciton operator should not be used if the line is empty. 
	- Extending cin and cout
		- By default, a programmers-defined C++ class does not work with cin and cout
		- The functionality of cin and cout can be extended by implementing certain friend functions in the C++ class
		- Friend:
			- It lets a function that's defined outside the class access private members directly as if it were inside the class. 
			- Cin >>: friend istream& operator>>
			- Cout <<: friend ostream& operator>>
