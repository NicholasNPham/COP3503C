MIDTERM REVIEW QUESTIONS

WHAT DOES THE NEW KEYWORD DO?
- The "new" keyword dynamically allocates memory on the heap at runtime and returns the memory address of that newly allocated memory, which is stored in a pointer.
- For example:
	
	int* ptr = new int;

WHAT DOES THE DELETE KEYWORD DO?
- The "delete keyword frees up memory that was dynamically allocated on the heap with "new".
- After calling "delete", you should always set the pointer to "nullptr" to prevent it from becoming a dangling pointer
- For example:

	delete ptr;
	ptr = nullptr;

- You cannot call "delete" on stack variables, only on memory that was allocated with "new".

WHAT IS THE PURPOSE OF USING BRACKETS WITH NEW AND DELETE:
- Brackets are used when dynamically allocating or freeing arrays on the heap.
- For example, 

	int* arr = new int[10]
	delete[] arr;

- "new" with brackets allocates enough memory for the entire array on the heap.
- "delete" with brackets tells the compiler to free all of that allocated memory. 
- If you use delete without brackets on the array, only the first element gets freed and the rest remains on the heap, causing a memory leak.

WHAT IS A MEMORY LEAK.
- A memory leak occurs when dynamically allocated memory on the heap is never freed with delete.
- Overtime the program keeps consuming memory wihtout releasing it back to the system, which can cause the program to crash or slow the entire system down.
- Memory leaks are silent, the compiler won't warn you, they only show up at runtime.
- This is why every "new" should always have a corresponding "delete"

WHAT IS A DEEP COPY COMPARED TO A SHALLOW COPY.
- a "SHALLOW" copy copies the pointer itself, meaning both the original and the copy point to the same memory address. 
- This is dangerous because if one object gets destroyed and calls "delete" on that memory, the other pointer becomes a dangling pointer.
- If the second object then tries to "delete" the same memory, it causes undefined behavior and can crash the program

- A "deep" copy allocates brand new memory on the heap and copies the actual data over element byt element, so the original and the copy are completely independent.
- Destroying one has no effect on the other
- For example

	int* copy = new int[10];
	for (int i = 0; i < 10; i++)
	{
		copy[i] = original[i];
	}
